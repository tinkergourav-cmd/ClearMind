# Preview Mode guard hardening across mutation paths

This change adds `if (isPreviewMode) return;` guards to 16 mutation functions and 4 inline connection-creation handlers in App.jsx that were previously unprotected. The direct bug fix is on edge Ctrl+Click deletion (now gated with `&& !isPreviewMode`). The approach is mechanical and consistent: every function that modifies project state gets an early return when in preview mode, matching the existing pattern already established elsewhere in the file.

**Watch for:** The multi-select floating action bar's Delete, Duplicate, and Connect buttons execute mutations inline without any `isPreviewMode` check inside the closure â€” they rely solely on being conditionally rendered (likely). `bringToFront`/`sendToBack` in the mobile sheet remain reachable in preview mode if z-order is persisted state (possible).

## High-level view

Sixteen named functions â€” covering edges, pins, tasks, task groups, canvas image drops, and partial import â€” now check `isPreviewMode` at the top. Four inline `onPointerUp` handlers that create connections also bail when in preview mode, and the edge Ctrl+Click handler now gates the call to `removeEdge` with `&& !isPreviewMode`, giving two layers of protection. The `cutNode` and `cutGroup` dependency arrays are fixed to include `isPreviewMode`.

The multi-select action bar (`selectedNodeIds.length > 0 && !isPreviewMode`) hides mutation buttons via a render condition, but the inline onClick handlers call `takeSnapshot()` and `updateActiveWorkspace()` directly without checking `isPreviewMode`. If a future refactor renders the bar while in preview mode, data mutates. This is the defense-in-depth gap the PR's own philosophy was meant to close.

The `cutNode` and `cutGroup` guards prevent a subtle interaction: `pasteNode` checks `clipData.action === 'cut'` and deletes the source node on paste. Blocking the cut in preview mode prevents a user from unknowingly staging a destructive paste for later.

## Multi-select action bar defense-in-depth gap

At line ~7117, the action bar renders only when `selectedNodeIds.length > 0 && !isPreviewMode`. Inside that bar:

```jsx
<button onClick={() => { takeSnapshot(); /* ...delete mutation... */ }}>Delete</button>
<button onClick={() => { takeSnapshot(); /* ...duplicate mutation... */ }}>Duplicate</button>
<button onClick={() => { /* ...connect mutation... */ }}>Connect</button>
```

None of these closures check `isPreviewMode`. The UI-level conditional rendering is the sole protection. This contradicts the PR's stated architecture requirement: "Every mutation/action should first check the global `appMode`." If the render condition is ever refactored (e.g., showing a disabled bar with tooltips), the mutation paths are exposed.

## `cutNode`/`cutGroup` guard justification

These functions only write to `localStorage`, not React state:

```jsx
const cutNode = useCallback((nodeId) => {
    if (isPreviewMode) return;
    // ...writes to localStorage only, no state mutation...
}, [nodes, activeTab, isPreviewMode]);
```

But `pasteNode` checks `clipData.action === 'cut'` and, on paste, deletes the source node from workspace state. If `cutNode` could run in preview mode, switching to edit mode and pasting would delete the original node without the user realizing the cut was staged in preview mode. The guard prevents this cross-mode hazard.

## Connection-creation handlers

The four inline `onPointerUp` connection-creation handlers all clean up UI state (`setConnecting(null)`, `setConnectHoverNodeId(null)`) before returning, so the connecting visual doesn't get stuck when the guard fires. The edge Ctrl+Click handler gets defense-in-depth: the inline condition `&& !isPreviewMode` prevents calling `removeEdge`, and `removeEdge` itself now also has an internal guard.

## Test coverage absence

No test framework is configured. The guard count went from 44 to 60 as measured by grep, confirming the additions, but no automated assertion prevents regressions. A single integration test simulating mutations in preview mode and asserting no state change would cover all guards at once.
